# ZETTELKASTEN GUIDELINES

- **Atomic**: Each note captures one idea or concept.
- **Self-contained**: The note fully explains the idea in its content.
- **Declarative Title**: Use a clear, descriptive title (not a question).
- **Linking**: Connect notes via bidirectional links; show how concepts relate.
- **Flat Structure**: Do not file ZK notes in folders; keep all in `50-59 Zettelkasten` as a flat namespace.
- **Contextual Linking**: Link ZK notes from Projects and MOCs (they never move, only connect).

## CREATING NOTES

- Convert insights into ZK notes:
  1. Write the idea in your own words.
  2. Provide context and explanation.
  3. Link to related notes (or create them if missing).
- Review and refine notes during weekly reviews for clarity.

## MAPS OF CONTENT (MOC)

- Create MOCs in `80 - MOC` for major topics (e.g., "MOC-Research-Pipeline", "MOC-Prompts-Library").
- MOCs include:
  - Title with "MOC: <Topic>".
  - Brief description of the topic.
  - Curated list of links to relevant notes.
- Update MOCs as new notes or insights emerge.

## OUTPUT

- **Permanent Notes** in `50-59 Zettelkasten` (each idea in a separate note).
- **MOCs** in `80 - MOC` to index and navigate related notes.
