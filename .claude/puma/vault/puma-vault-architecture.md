# VAULT ARCHITECTURE & ORGANIZATION

Follow PARA + Johnny Decimal and GTD in the PUMA vault:

- **10-19 Projects**: Active project work (plans, experiments, code).
- **20-29 Areas**: Ongoing responsibilities (research, dev, maintenance).
- **30-39 Resources**: External knowledge (papers, tools, data).
- **40-49 Archive**: Completed or deprecated projects.
- **50-59 Zettelkasten**: Atomic, permanent notes (concepts, insights).

## KEY RULES

- **Never** put permanent Zettelkasten notes or literature notes inside Project folders.
- Literature notes (paper summaries) belong in Resources.
- Permanent notes (insights, concepts) belong in Zettelkasten.
- Use **Johnny Decimal** in folder names for sortable structure (e.g., `10-1 Project-Alpha`).
- GTD tasks should be captured inline within project notes and queried globally with the Tasks plugin.

## LINKING

- Use bidirectional links liberally:
  - Link Projects to relevant Zettelkasten notes.
  - Link Resources (literature) to Permanent notes summarizing their insights.
  - Use Maps of Content (MOCs) to group related notes.

## PLUGINS & TOOLS

- **Dataview**: Query vault contents (tasks, notes).
- **Templater**: For consistent note creation (use provided templates).
- **Zotero Integration**: Manage references and citations.
- **Kanban**: For sprint/board visualization.
- **Excalidraw**: For diagrams.
- **Tasks plugin**: Global task tracking.
- **Obsidian Git**: Version control of vault.
- **Pandoc**: Export notes to documents (e.g., for thesis).
