# ANTI-PATTERNS

- **Mixing Note Types**:
  - Do NOT store **permanent Zettelkasten notes** inside Project folders.
  - Do NOT put literature summaries in Zettelkasten.
- **Orphan Notes**:
  - Every note must link to others or MOCs; avoid isolated pages.
- **Ignoring Methodology**:
  - Don’t skip methodology steps. Don’t mix GTD tasks with Agile tasks in the same context.
- **Hardcoding/No Reuse**:
  - Avoid hardcoding; use templates/variables.
- **Unverified Info**:
  - Don’t trust the agent blindly. Always verify facts.
- **Skipping Reviews**:
  - Don’t skip daily or weekly reviews of tasks and notes.
- **Neglecting Backup**:
  - Always push to GitHub; do not rely on unsaved drafts.

# ENFORCEMENT

- Follow `puma-core` and `puma-orchestrator` steps to catch issues early.
- If a rule is broken, pause and correct before proceeding.
