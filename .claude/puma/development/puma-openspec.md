# OPENSPEC USAGE

- **OpenSpec**: A structured YAML/Markdown format for specs (APIs, features).
- Use OpenSpec to:
  - Define agent interfaces, API endpoints, data schemas.
  - Link spec definitions to code or documentation.
- Store OpenSpec files in `40-49 Projects/41.6 Specs`.
- Example: Define agent prompt formats or evaluation criteria in OpenSpec.

## OUTPUT

- **OpenSpec YAML/MD** files for system components.
- Document each entry with descriptions and examples.
