# BMAD (Block-Methodical Adaptive Design)

- Divide the system into independent **modules/agents**, each with a clear role.
- Example roles:
  - **Architect** (advisor): Defines specs and oversight.
  - **Developer** (student): Implements code and experiments.
  - **Analyst/QC**: Tests and evaluates results.
- Treat each agent as a separate component with inputs, outputs, and responsibilities.
- Use BMAD’s iterative cycles aligned with project sprints.

## ARTIFACTS

- **BMAD Agent Roster**: List of agents, roles, responsibilities (`40-49 Projects/41.9 BMAD-Agents/BMAD-Agent-Roster.md`).
- **BMAD PRD**: High-level spec for each agent (`BMAD-PRD-PUMA.md`).
- Store these in the vault for team alignment.

## COLLABORATION

- Map BMAD roles to people: Advisor as Architect, Student as Lead Developer.
- Facilitate reviews: e.g., Architect reviews Developer's outputs.
