# SPEC KIT TOOLS

- **Spec Kit**: A collection of templates for technical docs.
- Use these templates in `40-49 Projects/41.6 Specs`:
  - **Constitution.md**: Project mission, vision, non-negotiable rules.
  - **Architecture.md**: System components, agent workflows, diagrams.
  - **UserStory.md**: Define scenarios (e.g., "Triage an issue with an AI agent").
- Link Spec Kit docs to tasks and issues.
- Treat `Constitution.md` as the project's guiding principles.

## OUTPUT

- **Filled Spec Kit templates** in the vault.
- Cross-links between specs and core PUMA rules.
