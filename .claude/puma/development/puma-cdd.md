# CONTEXT-DRIVEN DEVELOPMENT

- **Analyze Use Cases**: Identify real-world scenarios and user needs in project management (e.g., how students escalate issues).
- **Stakeholder Engagement**: Consider the perspective of advisors, developers, or target users.
- **Prototyping**: Build quick agent prototypes to test assumptions.
- **Documentation**: Record context scenarios and user stories in project notes *before* coding.
