# AGILE SCRUM/KANBAN

- Divide work into **sprints** (see `40-49 Projects/41.8 Sprint-Boards`).
- Use a **Kanban board** to track tasks (e.g., in `96 Kanban/Sprint-02-Board.md`).
- Follow Scrum events:
  - **Sprint Planning**: Define goals at start of sprint.
  - **Daily Standup**: (Record in daily notes).
  - **Sprint Review/Retrospective**: Reflect each iteration (use `Weekly-Review` template).
- Track:
  - **Product Backlog**: Features/stories (in project notes or separate doc).
  - **Sprint Backlog**: Tasks (using Tasks plugin or kanban).
- Continuously refine requirements and specs based on feedback.

## TOOLS

- **Kanban** plugin or board template (`Template-Sprint-Board.md`).
- **Tasks** plugin: Capture next actions globally.
- **Daily/Weekly Templates**: For consistent reviews.
