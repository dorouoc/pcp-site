🗺️ What is this vault?

This vault is the unified knowledge management system for the PUMA project. It integrates six complementary methodological systems into a single coherent workspace:

Layer System Purpose
Navigation MOC (Map of Content) Top-level orientation maps
Productivity GTD (Getting Things Done) Task & commitment management
Organisation PARA Projects · Areas · Resources · Archive
Knowledge Zettelkasten Atomic, permanent, linked ideas
Numbering Johnny Decimal Unique IDs for every note and folder
Engineering SDD / BDD / BMAD Spec-first development artefacts

Research frameworks integrated: EBSE+SLR/PRISMA · DSR · Grounded Theory · MIT Student Method · RCOIF · EGI · AMI · DRCA · IIPR · CoT · Few-Shot · Zero-Shot CoT · CDD · Agent OS · Agent Prompt Engineering.

🏗️ Vault Structure (Johnny Decimal)

  • 00 - Meta → Templates, dashboards, plugin configs
  • 10 - Inbox → Fleeting notes, quick capture (process daily)
  • 20 - Literature → All source materials (papers, books, datasets, tools…)
  • 30 - Permanent → Evergreen Zettelkasten notes (concepts, methods, results)
  • 40 - Projects → Active project work (PUMA chapters, specs, experiments)
  • 50 - Areas → Ongoing responsibilities (research, writing, code, ethics)
  • 60 - Resources → Reusable assets (prompts, workflows, checklists, glossary)
  • 70 - Archive → Completed and deprecated material
  • 80 - MOC → Maps of Content and master indexes
  • 90 - GTD → Tasks, reviews, someday/maybe lists
